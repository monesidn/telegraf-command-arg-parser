Static number

  • number(tokens: TokenWithIndexes[], min?: undefined | number, max?: undefined | number, strict?: boolean, parser?: (Anonymous function)): ParserResult
    • Defined in common/parsers.ts:29

    Parse a number consuming one or more tokens.

    Parameters

    • tokens: TokenWithIndexes[]

      Unparsed tokens.

    • Optional min: undefined | number

      Optional lower bound. If the parsed number is below the bound an error is returned.

    • Optional max: undefined | number

      Optional upper bound. If the parsed number is below the bound an error is returned.

    • Default value strict: boolean = false

      If set to false the parser will ignore some common mistake like (but not limited to):

      • a space between sign and number, commonly inserted by phone keyboards (eg "+ 5", "- 100"...)
      • one or more spaces between digits (eg 1 000 000 will be parsed as 1000000) When not in strict mode the parser will try to consume all subsequent tokens that form a valid number. Obviously strict parsing is faster but will degrade user experience.
    • Default value parser: (Anonymous function) = (i: string) => +i

      To parse numbers supporting different locales pass here a parser bound to the right locale. I suggest using a globalize-generated number parser. By default (and to avoid extra dependancies) the parser will use (i) => +i; that rely on javascript runtime settings.

    Returns ParserResult

    an object with a ParsedArgument and the remaining tokens.
